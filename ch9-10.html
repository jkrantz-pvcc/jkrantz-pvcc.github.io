Chapter 9: Introduction to High Level Languages

List four disadvantages of using assembly languages:

  1. the programmer must manually manage the movement of data items between and among memory locations and registers
  2. the programmer must take a microscopic view of a task, breaking it down into tiny subtask that direct what is going on in individual memory locations and registers
  3. assembly language program is machine specific
  4. statements are not natural-language-like

List four expectations progrmamer have for writing programs in a high-level language ("third generation" language):

  1. programmers need not manage the details of the movement of data items within memory or pay and attention to exactly where those items are stored
  2. programmers can take a macroscopic view of task, thinking at a higher level of problem solving, primitive operations used as building blocks in algorithm construction can be larger
  3. programs are portable rather than machine specific
  4. programming statements are closer to natural language and use standard mathematical notation

Briefly describe or define:

  1. object code- machine language intructions
  2. source code- high-level language intructions
  3. procedural language- programming language in which programs consist of sequences of statements that manipulate data items
  4. Integrated Development Environment (IDE)- a collection of programs that support software development, such as debuggers, editors, toolkits, and libraries, that lets programmers perform several tasks within the shell of a single applicatio
  5. software development life cycle- the overall sequence of steps needed to complete a large-scale software project
  6. waterfall model of software development- traditional model of software development that follows a series of steps
  7. agile model of software development- software development that emphasizes a flexible and ready response to meet a shifting target

Describe each of these components of the software development life cycle:

  1. feasibility study- evaluates a proposed project and compares the costs and benefits of various solutions
  2. problem specification- developing a clear, concise, and unambiguous statement of the exact problem the software is to solve
  3. program design- plans the structure of the software to be written
  4. algorith selection- chosing or developing the algorithm for the various subtasks
  5. coding- translating the detailed designs into computer code
  6. debugging- locating and correcting program errors
  7. testing- making sure the program produces correct answers and on all data
  8. verification- prove that if the input data to a program satisfies certain conditions, then, after the program has been run on this data, the output data satisfies certain other conditions
  9. documentation- all of the written and online material that makes a program understandable
