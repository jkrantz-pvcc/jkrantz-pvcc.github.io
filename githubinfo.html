All About GitHub

  • Q. What is a version control system?
    A. a software that manages project changes, allowing developers to store versions, compare updates, and revert when necessary
  • Q. What is Git?
    A. a distributed version control system
  • Q. What is GitHub?
    A. a proprietary developer platform that allows developers to creat, store, manage, and share code
  • Q. What is the diiference between Git and GitHub?
    A. git is free and is installed locally. github is a pay for use online service built to run git in the cloud
  • Q. Who started GitHub and how was it started?
    A. Tom Preston-Werner, Chris Wanstrath, Scott Chacon, and P.J. Hyett. They identified that Git would benefit from a decentralized hosting system
  • Q. What company owns it now?
    A. microsoft
  • Q. How much does a GitHub account cost?
    A. free for individuals
  • Q. What is the Octocat?
    A. a mythical creature that blends the characteristics of an octopus and a cat

Git/GitHub terms:

  • repository- a place to store your code, files, and each files revision history
  • commit- saving changes made to a project's source code in a shared repository
  • fork- a new repository that shares code and visibility settings with the original "upstream" repository
  • push- used to upload local repository content to a remote repository
  • pull requests- a proposal to merge a set of changes from one branch into another
  • workflows- a configurable automated process that will run one or more jobs
  • issues- a lightweight issue tracking system mostly used for reporting bugs and requesting features
  • raw button- you can view or copy the raw content of a file without any styling
  • blame button- used to examine the contents of a file line by line and see when each line was last modified and who the author of the modification was
