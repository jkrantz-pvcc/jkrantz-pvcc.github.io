Von Neumann computer architecture, 4 major subsystems:

  1. memory
  2. input/output
  3. arithmetic/logic unit
  4. control unit

Von Neumann computer architecture, 3 characteristics:

  1. four major subsystems
  2. instructions are represented as binary values and stores in memory
  3. sequential execution of instructions

Computer Architecture Terms

  1. RAM: a memory structure in which each cell has an address and it takes the same amount of time to fetch or story any cell
  2. cache: special high speed memory unit that keeps a copy of memory cells with a high likelihood of access in the near future
  3. Principle of locality: The high likelihood that when a program fetches a piece of data or an instruction:
    1. it will access that same instruction or piece of data in the very near future
    2. it will sonn access the instructions or data that are located near that piece of data
  4. ROM: a memory structure that can only be accessed, not written into or changed
  5. memory cell size: number of bits in a single memory cell
  6. memory cell address: unique numeric identifier for a memory cell
  7. maximum memory size: max amount of memory that a computer can physically hold, determined by the size of the address field
  8. register: special high speed storage cell that holds the operands of an arithmetic operation and that, when the operation is complete, holds its result
  9. fetch: access the contents of an address and return those contents as the result of the operation
  10. store: store the specified value into the memory cell specified by address
  11. memory access time: the time it takes to fetch or store the contents of a single memory cell
  12. DASD: a mass storage device in which each unit of information is associated with a unique address, but the time to access each piece of information may not be the same
  13. Arithmetic and Logic Unit (ALU): the subsystem that performs mathmatical and locgical operations
  14. Control Unit: the subsystem that fetches and executes instructions stored in the memory of the computer

Chapter 6:

Four things the interface between the computer and the hardware does:

  1. hides from the user the messy details of the underlying hardware
  2. presents information about what is happening in a way that does not require in depth knowledge of the internal structure of the system
  3. allow easy user access to the resources available on this computer
  4. prevents accidental or intentional damage to hardware, programs, and data

Responsibilites of systems software:

  1. hides the complex and unimportant details of the internal structure of the Von Neumannn architecture
  2. presents important information to the user in a way that is easy to understand
  3. allows the user to access machine resources in a simple and efficient way
  4. provides a secure and safe environment in which to operate

Terms:

  1. systems software: collection of computer programs that manage the resources of a computer and facilitate access to those resources
  2. user interface: visual overview of the computer that is provided by the OS
  3. language services: allow you to write programs in a high level user oriented language rather than machine language and to execute them easly and efficiently
  4. memory managers: allocate memory space for programs and data and retrieve this memory space when its no longer needed
  5. information managers: handle the organization, storage, and retrieval of information on storage devices such as hard drives and dvds
  6. i/o managers: allow you to easily and efficiently use the many different types of input and output devices that exist
  7. scheduler: keeps a list of programs ready to run on the processor and selects the one that will execute next
  8. utilities: collections of library routines that provide a wide range of useful services either to a user or to other system routines
  9. assembly language: a low level programming language that maps 1:1 into machine language

Functions of an Operating System:

  1. user interface management
  2. control of access to the system and to data files
  3. program scheduling and activation
  4. efficient resource allocation
  5. deadlock detection and error detection
